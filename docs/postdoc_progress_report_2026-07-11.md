# “一笔成章”智能书法辅助系统：模型模块阶段汇报

汇报人：刘小凡  
汇报时间：2026 年 7 月 11 日  
汇报对象：博士后老师  
项目定位：大学生创新训练项目；目标是在完成系统原型的基础上，沉淀一篇方法与实验可信的论文。

## 一、项目要解决的问题

面向初学者书写汉字与临摹书法时的反馈需求，系统接收一张完整汉字 RGB 图像，自动识别其笔画结构，并为后续的笔画顺序、结构位置、书写质量与目标字体风格评价提供可解释的中间表示。

当前模型模块的核心任务不是把汉字分成互斥类别，而是预测六张可重叠的结构概率图：

```text
vec1, vec2, vec3, vec4, vec5, keypoint
```

其中五个 `vec` 通道表示项目既有标签体系中的笔画方向/类别，`keypoint` 表示起收笔等关键区域。交叉处可能同时属于多个方向通道，因此采用独立 Sigmoid 多标签分割，而不是 Softmax 单标签分割。

## 二、拟形成的研究问题

本项目后续可围绕以下三个问题组织论文，而不是简单罗列“用了哪些大模型”。

### RQ1：预训练 Transformer 能否改善复杂汉字的结构分割？

在固定且无泄漏的数据划分下，验证 ImageNet 预训练的 SegFormer-B2 是否能比 U-Net 更好地处理：

- 笔画交叉和方向重叠；
- 相邻笔画粘连；
- 起笔、收笔和点画等小目标；
- 粗细变化与书写风格差异。

核心假设：SegFormer 的全局建模与多尺度表征可提升五方向通道的 Macro Dice，以及关键点 F1，且提升主要体现在困难样本上。

### RQ2：小数据条件下，SAM2 应如何为结构分割提供价值？

现有整字样本规模不足以支持把 SAM2 作为唯一端到端模型。拟将它定位为受控的辅助教师：以已有单笔画标签自动产生点/框/mask prompt，构造笔画实例任务，用于边界细化、伪标签候选和困难案例分析。

核心假设：在不做大规模全量微调的前提下，SAM2 的实例边界信息可改善交叉和端点区域的 Boundary F1；若不能稳定增益，则保留为标注辅助工具，而不作为最终在线主模型。

### RQ3：如何支持“目标字体选择”下的可解释反馈？

软件拟增加赵孟頫楷体、欧阳询楷体等目标字体选择功能。模型不宜为每种书体训练一套独立分割网络，而应采用：

```text
通用结构分割 + target_style_id / 参考字条件 + 风格差异评价
```

7 月先完成通用结构分割；风格条件数据、参考字库和风格评价头在 8 月构建。该设计可以避免风格样本不足导致的过拟合，也能保留后续扩展新书体的能力。

## 三、总体技术路线

### 1. 数据层

输入为完整汉字 RGB 图像，经等比例填充后统一到 `512×512`。标签保持与旧项目兼容的六通道 schema，并在 manifest 中记录样本、字符、来源/书写者、标签路径和数据版本。

数据划分按书写者或来源分组；现有旧数据缺少可靠身份信息时，暂按固定样本编号分组，避免同一编号样本同时进入训练和测试。

### 2. 云端高精度主线

以 SegFormer-B2 作为 7 月的主模型：

```text
RGB 汉字图像
  → SegFormer-B2 预训练编码器
  → 六通道多标签解码头
  → Sigmoid 概率图 [H, W, 6]
  → 通道独立阈值
  → 五方向掩码 + 关键点掩码
  → 结构化反馈/后续风格评价
```

训练损失拟采用：五方向通道“类别加权 BCE + Dice”，关键点通道“Focal + Dice”，并由标签计算边界监督，加入权重为 `0.2` 的 Boundary Loss。训练使用 AdamW、混合精度、编码器/解码器分层学习率、余弦退火和早停。

### 3. SAM2 受控辅助分支

SAM2 不取代 SegFormer-B2，而在离线训练环节承担：

- 用单笔画标注自动构造实例 prompt；
- 生成边界细化或伪标签候选；
- 分析困难集中的交叉、粘连、端点案例；
- 为后续蒸馏提供可能的边界教师信号。

实验止损点：使用相同数据划分与指标；若截至 7 月 25 日，SAM2 不能超过 SegFormer 的 Macro Dice 或关键点 F1，或不能稳定改善困难集 Boundary F1，则停止扩大该分支。

### 4. 云端—端侧双模型规划

| 层级 | 角色 | 当前状态 |
| --- | --- | --- |
| 云端教师 | SegFormer-B2 + SAM2 辅助 + 后续风格评价 | 7 月主线 |
| 端侧学生 | SegFormer-B0 或 MobileNetV3 + FPN | 8 月蒸馏、量化与 ONNX 部署 |

这样既保留云端的高精度与可解释性，也能为软件侧低时延体验预留路径。

## 四、当前进展与数据现状

## 五、目前发现的关键风险

### 1. 数据量与数据质量

840 个完整整字样本对训练 Transformer 和 SAM2 都偏少。54 个异常样本尚未完成复核，若存在系统性标签问题，会直接影响模型结论。

应对：优先完成异常样本决策；新增数据优先覆盖交叉、粘连、端点和模型低分字符；新增样本利用逐笔采集自动生成标签，并由人工抽检至少 20%。

### 2. 基线尚未建立

旧项目报告中的约 85.2% 自定义准确率不可直接作为新基线，因为数据划分与评测定义不够严格。

应对：以当前固定划分重新训练 U-Net，统一报告方向通道 Macro Dice、Macro IoU、逐类 Precision/Recall、关键点 F1、Boundary F1 和困难集指标。

### 3. 工程实现与实验设计的一致性

当前训练入口可以运行基础多标签训练，但尚未完整实现计划中的类别加权、Boundary Loss、分层学习率和余弦退火。若不先补齐，将无法证明性能提升来自完整方法设计。

应对：先完成小规模 smoke test，再启动 U-Net 正式基线与 SegFormer 消融；所有正式候选模型使用 3 个随机种子。

### 4. 字体选择功能的研究边界

仅凭“选择赵孟頫楷体”不能直接得出风格评价结论，必须有该书体的参考字与可解释评价规则。

应对：论文第一阶段聚焦“通用笔画结构分割”；字体条件化反馈作为系统扩展与后续实验，不与 7 月的核心分割结论混淆。

## 六、实验设计与验收标准

### 对照与消融

| 实验 | 目的 |
| --- | --- |
| U-Net 重测 | 可信基线 |
| SegFormer-B2：基础损失 | 验证预训练 Transformer 主干 |
| SegFormer-B2：+ 关键点加权 | 验证小目标关键点优化 |
| SegFormer-B2：+ 边界损失 | 验证交叉/端点边界增益 |
| SAM2 辅助 | 验证实例边界教师是否带来稳定增益 |

### 指标

- `vec1–vec5`：Macro Dice、Macro IoU、逐类 Precision / Recall；
- `keypoint`：F1；
- 边界：Boundary F1；
- 困难集：Macro Dice、Boundary F1 与可视化失败案例；
- 最终候选：三个随机种子的均值与标准差。

### 7 月主体验收目标

- SegFormer-B2 的五方向 Macro Dice 比可信 U-Net 基线提高至少 3 个百分点；
- 关键点 F1 提高至少 5 个百分点，任一主要方向通道下降不超过 2 个百分点；
- 困难集 Macro Dice 提高至少 5 个百分点；
- 训练、评测、单图推理可在干净环境一条命令运行；
- 30 个代表性结果经过书法相关成员人工复核，无系统性错分。

## 七、论文可能的贡献表述（需实验验证）

以下不是既成结论，而是目前希望通过实验验证的贡献方向：

1. 提出面向完整汉字的“方向笔画 + 关键点”六通道重叠结构表征，兼顾交叉笔画的多标签归属与起收笔区域。
2. 提出小样本条件下的“SegFormer 主分割 + SAM2 实例边界辅助”策略，并通过严格止损实验验证其有效边界。
3. 构建从通用结构分割到目标书体条件化反馈的云端—端侧可扩展系统路线，而非为每种字体训练独立模型。

论文表述应避免把“使用 SAM2”当作创新点；真正需要证明的是：该表示、训练策略和辅助机制是否在固定数据划分、困难集和多随机种子下带来稳定、可解释的提升。
